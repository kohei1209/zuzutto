# FastQC QCレポート 項目別解説

FastQC は FASTQ ファイルの品質を多角的に評価するツールです。
各項目には **PASS（緑）/ WARN（黄）/ FAIL（赤）** の判定がありますが、
RNA-seq データは特有の偏りがあるため、WARN/FAIL でも必ずしも問題ではありません。

---

## 1. Basic Statistics（基本統計）

| 項目 | 説明 |
|------|------|
| Filename | 解析したFASTQファイル名 |
| File type | ファイル形式（通常 Conventional base calls） |
| Encoding | クオリティスコアのエンコード方式（通常 Sanger / Illumina 1.9） |
| Total Sequences | リード総数 |
| Sequences flagged as poor quality | 低品質フラグがついたリード数 |
| Sequence length | リード長（範囲） |
| %GC | 全塩基に占めるGC含量 |

### 評価ポイント
- **Total Sequences**：RNA-seqでは通常 **1サンプルあたり 10M〜50M リード** が目安。極端に少ない（< 1M）場合は実験失敗の可能性。
- **%GC**：ヒトなら **45〜55%** 程度が正常。大きく外れる場合はコンタミネーションの疑い。

---

## 2. Per Base Sequence Quality（塩基位置別クオリティ）

リードの各塩基位置における**クオリティスコア（Phred score）の分布**を箱ひげ図で表示。

### Phred スコアの意味
| スコア | 塩基呼び誤り確率 | 精度 |
|--------|----------------|------|
| Q10 | 1/10（10%） | 90% |
| Q20 | 1/100（1%） | 99% |
| Q30 | 1/1000（0.1%） | 99.9% |
| Q40 | 1/10000（0.01%） | 99.99% |

### 判定基準（FastQC）
| 判定 | 条件 |
|------|------|
| PASS | 全位置の中央値が Q28 以上、25%タイルが Q20 以上 |
| WARN | 中央値が Q27 以下 または 25%タイルが Q20 以下 |
| FAIL | 中央値が Q20 以下 または 25%タイルが Q5 以下 |

### RNA-seq における注意点
- リード **末尾（3'側）** は品質が下がるのが正常（Illumina の仕様）
- Trim Galore でクオリティ Q20 以上を残すトリミングを行うことで解消
- トリミング**後**のレポートで全位置 Q20 以上を確認すること

---

## 3. Per Sequence Quality Scores（リード全体のクオリティ分布）

1本のリード全体の**平均クオリティスコアの分布**をヒストグラムで表示。

### 評価ポイント
- ピークが **Q28 以上** にあれば良好
- 低品質（Q20 以下）にも大きなピークがある場合は問題
- トリミング後に低品質リードのピークが消えることを確認

---

## 4. Per Base Sequence Content（塩基位置別塩基組成）

リードの各位置における **A/T/G/C の割合** を表示。

### 理想的なパターン
理論上、ランダムなシーケンスでは各位置で A≈T、G≈C になる。

### RNA-seq では WARN/FAIL が出やすい理由
- **リード先頭（1〜10 bp）**：Random hexamer プライマーによる配列バイアスで偏りが生じる（RNA-seq では正常）
- **アダプター配列**：トリミング前は 3' 端に偏りが見られる

### 対処法
- トリミング後もリード先頭の偏りが残る場合 → `--clip_R1 10 --clip_R2 10` で強制的に先頭をトリム
- それ以外の部分で大きな偏りがある場合は調査が必要

---

## 5. Per Sequence GC Content（GC含量分布）

リストごとの**GC含量のヒストグラム**と理論的な正規分布を比較表示。

### 評価ポイント
| 状態 | 意味 |
|------|------|
| 理論曲線と一致 | 正常 |
| シャープなピークが複数ある | 特定の配列のコンタミ（アダプター残存など） |
| ピークが大きくズレている | ゲノムコンタミ、rRNA コンタミ |
| 二峰性（ダブルピーク） | 異種コンタミまたは複数ゲノム由来 |

### RNA-seq の特性
- rRNA が除去されていない場合、GC 分布が偏ることがある（rRNA は GC リッチ）
- rRNA 除去（ribo-depletion）ライブラリでは特に確認が重要

---

## 6. Per Base N Content（N塩基の割合）

各塩基位置における**塩基が決定できなかった（N）割合**を表示。

### 評価ポイント
- N の割合が **1% を超える** 位置がある場合は問題
- リード先頭や末尾に N が集中している場合はトリミングで解消可能
- N が全体に散在する場合はシーケンサーの問題の可能性

---

## 7. Sequence Length Distribution（リード長分布）

リードの**長さの分布**を表示。

### 評価ポイント
- Illumina の場合、通常は**全リードが同一長**（例：150 bp）→ 単一バーのグラフ
- トリミング後は短いリードが増え、分布が広がる（正常）
- `--length 36` 以下のリードは除去される（本パイプラインの設定）

---

## 8. Sequence Duplication Levels（重複配列レベル）

同一配列がどれだけ重複して存在するかを示す。

### 評価ポイント
| 重複レベル | 解釈 |
|-----------|------|
| 低い（大部分が 1〜2 コピー） | 複雑なライブラリ、良好 |
| 高い（多くが 10 コピー以上） | 低入力量、PCR 過増幅の可能性 |

### RNA-seq での注意点
- RNA-seq は**高発現遺伝子のリードが本質的に重複する**ため、WARN/FAIL は正常
- Total Deduplicated Percentage が **< 20%** の場合は PCR 過増幅を疑う
- ATAC-seq や ChIP-seq と異なり、重複除去（deduplication）は RNA-seq では**通常行わない**

---

## 9. Overrepresented Sequences（過剰代表配列）

全リードの **0.1% 以上**を占める配列を一覧表示。

### 評価ポイント
| 検出される配列 | 意味と対処 |
|--------------|----------|
| アダプター配列 | トリミング未実施 → Trim Galore で除去 |
| rRNA 配列 | rRNA 除去の不十分さ → ribo-depletion 再検討 |
| ポリA配列 | 正常（mRNA 末端由来） |
| 未知配列 | BLAST 等で配列を同定して原因調査 |

---

## 10. Adapter Content（アダプター残存）

Illumina アダプター配列がリードに残存している割合を表示。

### 評価ポイント
- **トリミング前**：リード末尾にアダプターが検出されるのは正常（特に短いinsertサイズ）
- **トリミング後**：アダプター残存が **1% 以下** になっていれば良好
- Trim Galore はアダプターを自動検出して除去するため、通常は問題なく解消される

---

## チェックリスト（本パイプライン用）

### トリミング前
| 項目 | 確認内容 | 目安 |
|------|---------|------|
| Basic Statistics | Total Sequences | > 10M |
| Basic Statistics | %GC | 45〜55%（ヒト） |
| Per Base Sequence Quality | 中央値 | Q20 以上（末尾は低下可） |
| Per Sequence GC Content | 分布の形状 | 単峰性、理論曲線に近い |
| Per Base N Content | N の割合 | < 1% |
| Overrepresented Sequences | コンタミ配列の有無 | なし |

### トリミング後
| 項目 | 確認内容 | 目安 |
|------|---------|------|
| Per Base Sequence Quality | 全位置の中央値 | Q20 以上 |
| Adapter Content | アダプター残存率 | < 1% |
| Sequence Length Distribution | 最短リード | ≥ 36 bp |

## MultiQC でのまとめ確認

MultiQC は複数サンプルの FastQC 結果を1つの HTML にまとめます。

### 特に注目するパネル
1. **General Statistics**：全サンプルのリード数・GC・重複率を一覧確認
2. **FastQC: Per Sequence Quality Scores**：サンプル間でのクオリティばらつき
3. **FastQC: Per Base Sequence Content**：サンプル間で一貫した偏りがないか
4. **FastQC: Adapter Content**：トリミング後に全サンプルでアダプターが除去されているか

サンプル間で**大きな差異**があるサンプルは解析から除外するか、再シーケンスを検討してください。
